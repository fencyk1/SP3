Explanation

The purpose of this test was to check if the linker would be able to catch errors in the number of text records when the actual number is greater than the number given in the header record, if a missing linking record error would be thrown, and if a duplicate header record error could be thrown.

Results

The anticipated result was that the linker would throw each aforementioned error with the majority coming in the latter portion of the report. The actual result was an array index out of bounds error in the linker, causing it to abort immediately and give no user report or load module.

Our assessment was that this came from the linking record coming after the end record in the object file. thus, we tried moving it to a position before the end record, and the linker ran successfully. Results of the second run are shown below. Linker errors are noted in the errata report.
