Data Element Dictionary

This page contains all of the variables used in all of our classes.

Data Elements
Element Name Element Type Class Existed In Purpose
errorList ArrayList of ErrorData ErrorOut errorList contains all of the ErrorData objects for searching through, outputting, and maintaining.
counter int ErrorOut counter exists as a variable for iteration purposes, incremented each time a while loop loops.
exists boolean ErrorOut exists is a boolean used to tell the ErrorOut object whether or not an erro exists at the line being checked.
line int ErrorOut line exists to hold the line number that an error exists on.
lineNumber int ErrorData lineNumber holds the line number that an error exists on in the representation of ErrorData.
errorCode int ErrorData errorCode contains the error code assigned to the error it exists in, as represented in our error table.
errorMessage String ErrorData errorMessage contains the message the user will see when he or she encounters a specific error
immutableCopy int ErrorData immutableCopy is used to return various private ints in order to maintain immutable code
directiveNameTable Properties DirectiveTable directiveNameTable is a table used to store the directive string and whether or not it impacts memory.
tableFileName File DirectiveTable tableFileName is the name of the file from which the directives table will be imported in the constructor that uses a file name.
lineTokens StringTokenizer DirectiveTable lineTokens is used to tokenize the string from the input file to get the directive name and the memoryUse boolean in a constructor.
input BufferedReader DirectiveTable input contains the input from file, i.e. from directives.tbl located at the root of the code's directory.
newLine String DirectiveTable newLine exists to take one line at a time from the file directives.tbl and add it into the program.
directive String DirectiveTable directive holds the directive found in directives.tbl used to move into directiveNameTable.
memoryUse String DirectiveTable memoryUse holds whether or not the directive impacts memory, and is added to directiveNameTable.
decimal int Converter decimal holds the decimal representation of a given number or ascii string.
hexOut String Converter hexOut holds the hexadecimal representation of a given number or ascii string.
counter int Converter counter is incremented after each successful trip through a while loop.
digit char Converter digit is used to hold a single digit of a hexadecimal or binary number in String format.
dec String Converter dec is used to hold the decimal representation of a number or ascii string.
binary byte array Converter binary is used to store the byte representation of ascii characters, and convert them into binary.
rep int Converter rep is used to store the decimal representation of a binary string.
currentBin String Converter currentBin is used as a medium for concatenation.
totalBin String Converter totalBin is used to hold the final/total binary value.
conversion int Converter conversion is used to convert a binary number into an integer.
instructionsTable InstructTable AssemblerMain instructionsTable is used to import and hold the valid instructions from instructions.tbl.
directivesTable DirectiveTable AssemblerMain directivesTable is used to import and hold the valid instructions from directives.tbl.
errorsTable ErrorTable AssemblerMain errorsTable is used to import and hold the error types, codes, and messages from error.tbl.
sourceCodeFile InSourceCode AssemblerMain sourceCodeFile is used to hold the original source code given by the user.
sourceFile File AssemblerMain sourceFile is used to contain the source code.
errorsFound ErrorOut AssemblerMain errorsFound holds all of the errors found in the source code.
tokenizer Tokenizer AssemblerMain tokenizer is used to tokenize through each line of source code.
parser Parser AssemblerMain parser is used to parse the source code.
endOfProgram boolean AssemblerMain endOfProgram holds whether or not the source code's program has terminated, true if terminated, false if running.
lineNum int AssemblerMain lineNum is the current line number of the program.
errorValue Properties ErrorTable errorValue contains the error code and error type of the errors in the error table file.
errorMessage Properties ErrorTable errorMessage contains the error code and error message of the errors in the error table file.
lineTokens StringTokenizer DirectiveTable lineTokens is used to tokenize the string from the input file to get the error code and error type in a constructor.
errorCodes Properties ErrorTable errorCodes contains the error code and error message in reverse order.
errorCode String ErrorTable errorCodeholds the value of the error code to be but into the error table.
errorType String ErrorTable errorCodeholds the value of the error type to be but into the error table.
input BufferedReader ErrorTable input gets input from the file error.tbl and loads in into scope of our program.
newLine String ErrorTable newLine contains a line from the source file error.tbl and adds it into the representation in our program.
source ArrayList of String InSourceCode source contains the source code line by line in an array.
input BufferedReader InSourceCode input gets input from the source file and loads it into the source array.
newLine String InSourceCode newLine contains the source code line by line after being fed by the buffered reader, for storing in the array.
instructionType Properties InstructTable instructionType contains the instruction name and the instruction type as represented in the instruction table.
instructionOpcode Properties InstructTable instructionOpcode contains the instruction name and opcode as represented in the instruction table.
instructionFunctionCode Properties InstructTable instructionFunctionCode contains the instruction name and function code as represented in the instruction table.
input BufferedReader InstructTable input gets input from the source file instructions.tbl.
newLine String InstructTable newLine holds the input from the buffered reader and stores the data in the instrution fields.
lineTokens StringTokenizer InstructTable lineTokens is used to tokenize the string from the input file to get the instruction type, opcode, function code and the instruction in a constructor.
instructType String InstructTable instructType holds the value of the type of instruction to be but into the instruction table.
instruction String InstructTable instruction holds the value of the instruction to be but into the instruction table.
functionCode String InstructTable functionCode holds the value of the function code to be but into the instruction table.
opcode String InstructTable opcode holds the value of the opcode to be but into the instruction table.
label String Symbol label represents the label of a symbol in the source code.
location int Symbol location holds the location in memory of the represented symbol.
length int Symbol length holds the length in bits of the symbol, the default being one word.
usage String Symbol usage represents how the symbol is used, be it label, execution point, etc.
symTable ArrayList of Symbols SymbolTable symTable contains all of the symbols in the symbol table.
label String SymbolTable label represents the label of a symbol in the source code.
counter int SymbolTable counter is incremented after one loop of a while loop, used to escape the loop.
innerCounter int SymbolTable innerCounter is incremented after one loop of a while loop, used to escape the loop..
exists boolean SymbolTable exists holds whether or not a specific symbol exists in the symbol table.
returnValue int SymbolTable returnValue contains the value, be it length or location, of a specific symbol to be returned.
sorter ArrayList of Symbols SymbolTable sorter sorts a SymbolTable based on the label name of the symbol.
currentSymb Symbol SymbolTable currentSymb holds the current symbol in a symbol table, used to output its data.
output String SymbolTable output holds the entire output of a symbol table, used to output the symbol table as text.
tokenArray ArrayList of String Tokenizer tokenArray is the array list that will be passed back to the method caller.
commentArray ArrayList of String Tokenizer commentArray is a temporary array list for storage of comment tokens.
spaceArray ArrayList of String Tokenizer spaceArray is a temporary array list for storage of space tokens.
commaArray ArrayList of String Tokenizer commaArray is a temporary array list for storage of comma tokens.
i, j, k int Tokenizer i, j, and k are temporary variables used for storing positions in the arrays
comma, space StringTokenizer Tokenizer comma and space are used to turn incoming lines of program code into tokens.
temp String Tokenizer temp is a string object used to store tokens.
delim String Tokenizer delim is a string object used to determine what is to be delimited in the tokenizer.
i int UserReport i is used to determine the positioning of output in the UserReport
errorEntry ErrorData UserReport errorEntry is used to hold new error messages.
out PrintWriter UserReport out is used to get input from the file directives.tbl
inDotText, inDotData, haveDotStart boolean SourceCodeParser These booleans are used to filter lines of tokens into which sections to parse.
prgmName Symbol SourceCodeParser prgmName is used to hold the values of the program name necessary to put into the symbol table.
locationCounter int SourceCodeParser this is used to keep track of the value of the location counter while parsing the program.
lcConverter, lcToHex String SourceCodeParser These are used to convert lc values for storing into the symbol table.
errorsFound ErrorOut SourceCodeParser Keeps working record of all errors discovered during parsing of the program.
programName String SourceCodeParser programName is used to hold the name of the program the user gives us.
syntaxError, invalidString, etc. ErrorData SourceCodeParser Used to put error info into the errorOut to store errors.
errors boolean SourceCodeParser an error flag for encoding purposes
converter, dickables Converter SourceCodeParser converter is used to convert various data to different representations.
stringHolder String SourceCodeParser A String object to hold the String data to be parsed.
parameter String SourceCodeParser parameter is the string representing an operand of an operation.
munchables, hexInteger int SourceCodeParser Used to check memory / numerical value for bounds errors.
imm int SourceCodeParser holds the value of immediate-value operands to parse.
toEncode String Encoder Used to store the value of the SAL560 program line to be encoded.
converter Converter Encoder Used to do the conversion to binary.
opCode, fnCode String Encoder holds the opcode / function code of the instruction to be encoded.
reg1, reg2, reg3, addr String Encoder hold the values of the operands to be encoded.
i int Encoder counting variable used during iteration.
skip int Encoder In the mem.skip operation, tells the object file to skip the needed number of line of code.
encoded String Encoder Used to hold the working value of the binary encoding of a SAL560 program line.
binCode ArrayList of Strings IntermediateFile Holds the values of the binary encodings to be outputted into the intermediate file.
out PrintWriter IntermediateFile out is used to print output to populate the intermediate file
i int IntermediateFile Counting variable used during iteration.
textLines, linkerLines ArrayList of type String ObjectFile used to store the values of the Text- or Linker-lines for the object file.
headerLine, endLine String ObjectFile used to store the text for the header / end line of the object file.
programLengthString String ObjectFile Used to create the value of the hex encoding for the program length.
prgmLoadPoint String ObjectFile used to store the loading address of the program for the current line.
totalRecords int ObjectFile used to hold the total number of records in the file, 1 for header 1 for end, then the rest.
