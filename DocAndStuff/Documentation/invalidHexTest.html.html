Explanation

The purpose of this test was to ensure that the linker would be able to catch errors in hexadecimal sections of the text, linking, end, and header records of an object file.

Results

the anticipated result was that the linker would throw errors for invalid hex formatting for hex input "0AE00e4G" [and change that line to be a nop], the hex input for the end record for total number of records, the header record's program length field, and the linking records address location field.

the unfortunate actual result was that the linker ran into a "Number Formatting Exception" while trying to build the linker load module for the object file. This in turn resulted in the linker immediately aborting all processes, which lead to no linker user report or load module being completed by the linker.

We assessed that the linker ran into an error while attempting to process the "FFGF" hex input of the fault text record, and replaced that input with a valid hex formatting. This resulted in the linker successfully being able to generate and complete both a linker user report and a load module. also to note, an extra error was unexpectedly detected in the linking record with the formatting of the TYPE field: "Program Name" instead of "Start." See results below.
